okCycle

Tiny, modular, flexible slideshow

Demonstration

Fading Transition with Autoplay UI

Stop the autoplay by hovering the slideshow. This example (and others) also show the 'caption', 'navigation'(prev/next buttons), 'pagination' and 'currentPage' UI. 

          
$("#example-1").okCycle({
  autoplay: true,
  transition:'fade', 
  ui: ['caption','navigation','pagination','currentPage']
});

Scrolling Transition with MouseWheel UI

You can also use your mousewheel on this example

            
$("#example-2").okCycle({
  transition:'scroll',
  ui: [
    'caption',
    'navigation',
    'pagination',
    'currentPage', 
    'mouseWheel'
  ]
});

The normal behavior is to use the elements data-caption attribute to provide captions. However, if the data-caption is prefixed with an octothorpe (#), it will be treated as a selector and the innerHTML for the corresponding element will be used.

Sliding Transition with Deferred Loading and Progress

          
$("#example-3").okCycle({
  transition:'slide', 
  ui: ['navigation','pagination','currentPage','progress'],
});

This example lazy loads images by setting the data-src attribute rather than the src attribute Image loading will be deferred until a given item is made active.

Important - you must use a data property to set the image source to prevent the browser from automatically downloading the image. This is defined by the dataAttribute option and be default is src. If you forget to do this, nothing bad will happen, it just won't be lazy loaded. 

          
// Where 'W' and 'H' are the width and height of your image
<img src='placeholder.png' data-src='/path/to/image' alt='' width='W' height='H' />

The progress UI makes use of image loading api function, progress and done, which wraps a jquery deferred. Note that callbacks are applied across the entire matched set, calling .progress or .done returns the API object, not a deferred, as you would normally expect. You can hook into them in the following manner: 

          
var api = $(selector).okCycle();

// Inside the callback 'this' is set to the slideshow
api.progress(function(data, image){
  // ... Do stuff
});

api.done(function(options){
  // ... Do stuff
});

Additionally it's possible to manually control the slideshow with the api 

          
var api = $(selector).okCycle();

api.pause(); // Stop autoplay
api.play();  // Resume autoplay
api.next();  // Move to the next slides
api.prev();  // Move to the previous slides
api.moveTo(index); // Move to a specific slide
