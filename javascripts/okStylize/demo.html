okStylize

Styling for unstyleable form elements: checkboxes, radios, selects and file inputs

okStylize can be used to style ANY input - though you probably shouldn't, because it is unnecessary!

          
// On every input in a form
$("form").okStylize();

// Or on an individual input
$("form :input").okStylize();

okStylize essentially does three things to inputs:

  1. Wraps the input in additional elements to assist with styling (customizable)
  2. Sets appropriate classes on the wrapped input regarding state (selected, hover, disabled etc.)
  3. Proxies events to the original input so changes to the UI are mirrored in the actual form input

Note that the styles are not part of the okcreate! framework, but are simply for demonstration.

Extending Fields

If you wish to add new styled inputs or alter the behavior of an existing wrapper, you must extend the $.okStylize.ui object. Each wrapper is a plain object with a number of properties that control how it is rendered and which events it responds to.

Let's take a look the implementation of the file wrapper: 

          
$.okStylize.ui = {
  file: {
    // This function should wrap the input in as much extra markup as needed
    // to achieve the look you're going for. This function should return
    // element that events should be bound to.
    setup: function(input,options){
      var html   = "<span class='filename'></span><span class='action'>"+opts.fileButtonHtml+"</span>",
          parent = input.wrap("<div/>").parent().append(html);

      if (input.prop("size")) input.prop("size", parent.width() / 10); 

      return parent;
    },
    // Determines if this wrapper should be applied to a given input. Should be a jQuery selector
    selector: ":file",
    // Determines which event will cause the update function to fire
    bind: 'change',
    // In this function you should apply any UI changes to the wrapper. In the
    // case of the file, if a file is selected, we want to show the filename
    // inside the input. Conversely, if no file is selected we want to clear the
    // file name. This function will be called both when the event is triggered
    // and immediately after setup is performed
    update: function(input,options){
      var filename  = input.val(),
          container = input.parent();

      if (filename === "") {
        filename = opts.fileDefaultHtml;
      } else {
        filename = filename.split(/[\/\\]+/);
        filename = filename[(filename.length - 1)];
      }

      container.find(".filename").text(filename);
    }
  }
};
