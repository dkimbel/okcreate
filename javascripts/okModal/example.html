okModal

Smart, easy and responsive modal windows.

Content Type Detection

okModal tries to take care of common use cases while getting out of your way if you want to do something crazy. The defautl invocation will intelligently determine the content type based on the triggering element. 

  
  // Call on the selector of the elements that will trigger a modal 
  // (see jquery.okModal.core.js for initialization options)
  $(".modal").okModal();

When the element's href points to an image, it is treated at an image.

  • Load a small image
  • Load an very wide image (Naturally wider than the viewport)
  • Load an very tall image (Naturally taller than the viewport)
  • Load a very big image (Naturally bigger in both dimensions than the viewport)

When the href points to a known video provider, it's treated as a embedded video. Only youtube and vimeo are supported by default, but it can be extended to support additional providers

  • Load a Video

When the href points to a known map provider, it's treated as a map. Only Google Maps, but again, it can be extended to support additional providers

  • Load a Map

If the href is an anchor, it is inferred to correspond to the id of an element somewhere in the page

  • Load In-page content

If it doesn't match any other matcher, it is assumed to be an get request to retrieve some content. Note this won't work unless this file accessed through a server

  • Load HTML

Aside from setting the dimensions in plugin initialzation, you can also specify dimensions in page on the element itself 

  
<a class='modal' data-height='500' data-width='500' href='...'>Link</a>

Extending Content Types

You can add additional content types by extending the OkModal.prototype.contentTypes object. 

  
$.extend(OkModal.prototype.contentTypes, {
  myContentType: {
    // The matcher should return a result if the given element matches the content type
    // What it returns is up to your implementation, but the result will be fed into 
    // the content function
    matcher: function(element){ return matcherResult; },
    // The content function should return a DOM Element, jQuery object or an HTML string
    content: function(element, matcherResult){ return html; }
  }
});

Transitions

okModal provides a few transitions that you can use to control how the content is shown on the screen.

Note that these transitions require the jquery.transit plugin to be loaded. Without it, the transitions will simply fade the modal in.

Fade In (Default) 
  
$(".modal").okModal();
Slide In (Top) 
  
$(".slide-top").okModal({ transition: 'slide' });
Slide In (Right) 
  
$(".slide-right").okModal({ transition: 'slide', direction: 'right'});
3D Flip (Horizontal) 
  
$(".flip-h").okModal({ transition: 'flip' });
3D Flip (Vertical) 
  
$(".flip-v").okModal({ transition: 'flip', vertical: true });
Super Scaled 
  
$(".super-scale").okModal({ transition: 'fade', scale: 2 });
Full Screen 
  
// Can be used with any transition. Just toggles the html element to overflow:hidden
$(".full").okModal({ dimensions: { height: 'natural', width: '100%' }, fullscreen: true, scale: 1 });

Adding Transitions

Similar to content types, you can add additinal transitions by extending OkModal.prototype.transitions 

  
$.extend(OkModal.prototype.transitions, {
  myTransition: {
    onOpen: function(modal, options){
      // Do some visual wizardy
    },
    onClose: function(modal, options){
      // Do some visual wizardy
    }
  }
});

// You can then use your fancy new transition by passing in the transition option
$(".modal").okModal({ transition: 'myTransition' });

UI

okModal comes with one prebuilt UI - a gallery. All the items that match the given selector will be treated as part of the same gallery. 

  
$("#gallery-example a").okModal({ ui: 'gallery', dimensions: { scale: 80 } });

Manual Usage

Instead of invoking okModal through $.fn.okModal, you can invoke it through $.okModal, which will allow you manually control the modal. There are three methods you should be concerned with open,close and destroy. Open and close show and hide the modal respectively, and destroy will remove the modal from the page and unbind events.

Click Me

Note that no events are bound for you in manual mode. 

  
var modal = $.okModal();

$("a.manual").click(function(e){
  e.preventDefault();

  // modal.open takes the content that you want to show
  modal.open("<div>Hello World!</div>");

  setTimeout(function(){
    modal.close();
  },2000);
});
