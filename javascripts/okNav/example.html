okNav

Demonstration

okNav provides abstractions over several different types of navigation interfaces. The basic requirements are that:

  1. The navigation interface must be made up of links contained within an element. By default, it expects a list of links.
  2. The anchors of the links must match the id of the content element that they correspond to.

While the example markup works, you're not purely limited to any particular arrangement and the content can exist anywhere in page. okNav does no styling for you, so don't be surprised if the tabs don't look like they do in the demo.

Tabs

This is the default user interface - if the ui option is omitted this is what will be used

            
$("#tabs").okNav(); // Called on the navigation

ScrollSpy

The ScrollSpy UI is for automatically updating nav targets based on scroll position. Scroll the area below the navbar and the menu will be updated based on your location. If differs from tabs as the siblings aren't hidden, and the tab active class is updated on scroll, rather than click.

If you want to target an element other than the body, you can supply a element which will be used as the scrollspy target.

Optionally, if you include jquery-scrollto, you can achieve a smooth scrolling effect by passing an options object for preventScroll. 

            
$("#scrollspy").okNav({
  ui: 'scrollSpy', 
  element: '#scrollspy-content',
  scroll: {
    duration: 600
  }
});

SelectNav

This one is a bit different than the others and has its own set of options (see the UI script). It converts list navigation into a select dropdown for better navigation on mobile devices. It can be used to intra-page or regular navigation. Navigation can be arbitrarily nested as well. 

            
$("#selectNav").okNav({ui: 'selectNav'});

  html.js-enabled ul#selectNav { 
    display: none; 
  }

  html.js-enabled select.selectNav { 
    display: block; 
  }
}
